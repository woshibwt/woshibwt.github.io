CS224N笔记-八-机器翻译，seq2seq与Attention

发表于 2020-07-15 | 分类于 CS224N |

  这一讲介绍了nlp领域中的一个重要分支即机器翻译，机器翻译是将源语言中的文字转换为对应目标语言文字的任务，早期的机器翻译都是人为的，随后发展出了依赖于统计信息的Statistical Machine Translation(SMT),之后又发展出利用神经网络使得准确度大幅改善的Neural Machine Translation(NMT),它依赖于seq2seq的模型架构，以及进一步将其改善的Attention机制。

Section 1:Pre-Neural Machine Translation

   机器翻译（MT）是将一个句子$x$从一种语言（源语言）转换到另一种语言（目标语言）的句子$y$的任务。

SMT

  SMT的核心思想是是从数据中学习概率模型。假设想要将一句法语句子$x$翻译成英文,我们希望找到对应最好的英语句子$y$，即找到$argmax{y}P(y|x)$。
  根据贝叶斯公式，这相当于在求$argmax{y}P(x|y)P(y)$（translation model）,其中$P(x|y)$负责模拟单词或词组该如何翻译，主要处理局部的词组的翻译;$P(y)$(language model)负责如何挑选这些单词或词组组成合理的目标语言中的句子。之前Language Model And RNN有提到如何学习语言模型，而对于translation model需要parallel data,即大量成对的法语、英语句子。
  SMT是一个非常巨大的研究领域，有非常多的重要细节，需要大量的人力来维护，对于每一对语言都要重复操作。NMT的出现极大地改善了这些问题。

Section2:Neural Machine Translation

Neural Machine Translation 与 Seq2Seq

  我们可以选择greedy decoding即每一步选取概率最大的单词将其作为下一步的decoder input


  greedy decoding带来的问题是当前的最大概率的单词未必对整个句子来说是最优的选择，但是由于无法撤销决定，我们没机会选择另一条整体最优的路径。
  解决这个问题的一个常用方法是beam search decoding,它的基本思想是在decoder的每一步，包括概率最大的单词，一共保存k个当前最有可能的翻译假设，其中k被称为beam size,通常在5到10之间。

NMT的优点：

  • 更好的性能
  • 单个神经网络端到端优化
  • 对所有语言采取相同的方法

NMT的缺点：

  • 可解释性较差
  • 很难控制

    BLEU

      常使用BLEU(Bilingual Evaluation Understudy)来评估MT的效果。BLEU将机器翻译和（一个或多个）进行比较，计算出一个它们相似度的分数。，它会对过于短的机器翻译加上惩罚。
      BLEU很有用，但并不完美，因为一个句子会有很多有效的方法来翻译。一个好的翻译可以得到一个糟糕的BLEU score,因为它与人工翻译的n-gram重复率较低。
